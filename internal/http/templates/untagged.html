Untagged Plugins

These WordPress plugins have a latest version on wordpress.org that isn't tagged in SVN. Some release exclusively via trunk, while others have older tags but not for the current version.

{{formatNumberComma .Total}} plugins affected ({{pct .Total .TotalPlugins}}% of all plugins)

WordPress officially recommends tagging every release in SVN and is working on disabling trunk as a release mechanism. Until then, these plugins can be installed via dev-trunk but can't be pinned to specific versions.

If you depend on one of these plugins, the best path forward is to ask the plugin author to tag their releases in SVN. It takes one command and makes the version available for Composer installs. See issue #15 for more details.

Install an untagged plugin via Composer

You can install any plugin from trunk using dev-trunk. Composer will pin to a specific SVN revision in your lock file.

composer require wp-plugin/example-plugin:dev-trunk
